The Balloon exploding factory

A VR mini serious game application to help research in upper limb (hand) motor function.

This application is downloadable on Sidequest here and the source code is on my GitHub here
The research paper for this project is published on my Researchgate here
Or you may watch the video demo:

This is a mini-game application/task to help research in upper limb (hand) motor function. It should be used with an Oculus Quest or Rift S as it utilizes hand tracking.

32 variables were recorded at every Fixed frame. The Fixed frame time used was 0.013s which correspond to a frequency of around 77 "Snapshots" of the hand and fingers state every second. Only data of the dominant hand, where the pump is attached, were captured. Speed and aperture length are calculated in real-time at the same rate.

I have developed a curve UI screen to display legal information and a virtual keyboard to allow the participants to interact

I build this virtual tablet as a live debug screen, showing all data being collected and check their accuracy and consistency

The hand pump has been designed using Blender and then imported into Unity.

Then capsule colliders is used to interact with the participant's hand to make it more realistic, which is very important in Psychology

Animator view of the avatar on unity.

The score and performance feedback are displayed on the virtual screen of the command panel

Augmented Reality Hand Tracking

A project to modify a VR headset Oculus Rift CV1 to an AR device with hand recognition using Unity and a CNN.

The research paper for this project is published and available on my Researchgate here
The poster of the project (see below) can be downloaded here

The aim of this project was to implement a stereo camera platform to support Augmented Reality for a Virtual reality headset. To be fully immersive, the frames captured by the cameras must match the characteristic requirements, such as field of view, of the headset device and make both the video and the virtual world aligned as much as possible to augment the real world. Hand recognition systems could create a natural interaction between human and computer device, so the user can control UI or any applications designed to do so.

Unity view, playing mode. Flask is used to stream a sequence of JPEG pictures, also known as Motion JPEG. This solution has the advantage of less latency, but meaning less quality too, because JPEG compression is not very fitted for streaming.

HCI and UI in VR issues

The asylum is a VR game in a 3D environment when a player can explore an abandoned Asylum. A usability test was carried to determine if the interface and UI are effective, intuitive as well as comfortable.

Investigation on the difficulty of HCI in the context of Virtual Reality and an attempt to advise some solutions (Teleportation, Curve UI, etc.) to VR and sick motion issues.

To avoid motion sickness, I have implemented two different ways of moving. Moving freely around by using the joystick. The walk speed is calibrated to slow with acceleration by pulling and holding the left trigger while you move. The left joystick function is to turn the camera at an angle view by 15° steps of your point of view. Then there is no fast motion. For people suffering of motion sickness, they can use the teleportation system. User can activate it by pushing the button X. A ray cast appear flowing from the right hand striking the ground right front of the hand, then the player can teleport to the place aimed and avoid any movement.

A fade system is implemented for the transition between scene (Here from the Curve UI at the begining of the game to the main scene), to avoid a loss of headtracking and a freeze of the last frame which is very uncomfortable and gives nausea

View of the fade system on the Unity scene

When the scene starts, to make the view comfortable, the menu panel appear straight front of the player with some explanation of how to play but also health warning.

A gaze pointer is visible when it over any clickable content. After closing the panel, users are able to reopen it whenever they want simply by pushing the standard “menu” button of the controller.

Smart Valve

A remotely operated valve system using microcontrollers and servo's.

We were briefed to create a remotely operated valve system using microcontrollers and servo's. The hardware would be internet connected and would be able to be monitored and controlled remotely and locally.

The system is designed to control an arduino based stepper motor system to open and close valves remotely from a web interface. The hardware automatically stores all open/close, test cases and errors to an SQL log which it then displays over on the tables page. We've also made use of charts to help visualise the data. My role was Embedded Systems Developer and Hardware Engineer

Part of our valves brief included a web interface to remotely control, monitor and test the systems. I utilised my knowledge of Javascript & bootstrap to help completing a fully functioning website, accessable on mobiles and browsers. The website made use of SQL to store individual valve data.
